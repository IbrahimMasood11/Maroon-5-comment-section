Dataset Overview

We will be using our project dataset which has 499 rows and 21 columns more than the required 200 rows for the project. The data is drawn from a YouTube video and reflects people's behaviour toward one another online.

There are three kinds of information such as numbers, categories, and true/false. This will help to do analyse the data in various perspective and forms chart pattern recognition.

Number columns include like count, reply count, and how many characters are in each comment. These numbers shows us how much people engage with the comments. For example, more likes or replies means to people who are more interested in what was posted.

Category columns include the ID number, author name, and parent ID. These help us identify each row and show which replies connect to which original posts.

True/False columnstells us whether something is a reply or an original post. This helps us separate regular posts from conversations between users.

We chose this dataset because it's good for analysing how people interact online. Having many rows (499) gives us reliable results, and having several number columns means we can do calculations and create charts. The data helps us spot trends in what gets the most likes and responses.

This data set is great for projects, since we can run descriptive stats on it, do performance visual charts and also apply basic machine learning. All of the data is public and does not include personal information, so its use is ethical.

Data Cleaning Summary

Fixing problems in the data beforehand improves our analysis. First, we reviewed the dataset to check for missing information, incorrect formats, and duplicate entries.

Missing Information: Some cells were empty, especially in the parent ID and unnamed columns. Parent ID are blank because most posts are original, not replies. The unnamed columns were almost entirely empty, so we removed them.

Correct Formats: We verified the numbers (likes, replies, character length) were formatted properly for calculations and charts.

Unusual Values: We found some extremely high numbers, like posts with many likes or very long text. These represent real user behavior, not errors, so we kept them to maintain accuracy.

This process ensures our statistics and visualisations are based on quality information.

Machine Learning Analysis:

Machine learning is a subcategory related to AI (Artificial Intelligence) that allows systems to learn patterns from data that it is fed without necessarily programming. There are three main subsets of ML (Machine Learning), this includes:

Supervised Learning: trains a model using a dataset where each input always comes with a known output. The goal is for the model to understand relationships between these input=output so it can begin to predict the output for new, unseen data.

A simple way to explain this is saying SL is like a student practicing with a workbook where every question has an answer key, the student then compares his answers to the answer key and corrects what’s wrong.

Unsupervised Learning: works by discovering hidden patterns in unlabelled data through clustering (grouping customers by behaviour) or dimensionality reduction (simplifying data while keeping important information). The model looks for patterns, groupings, or relationships all by itself, the model is not given a "correct answer" during training. It's often used for exploration, compression, or pattern discovery.

Reinforcement Learning: works by learning by interacting with a prebuilt environment. The most common examples we could find include game-playing agents (Chess, Go, Video games), robotics (learning to walk, grab objects, etc.), recommendation systems that adapt over time.

This shows that the Machine Learning workflow commonly involves data collection/processing, feature engineering, model training/validation, hyperparameter tuning, and deployment with continuous monitoring.

Limitations and Ethics

Machine learning systems offer great potential across many industries. However, they face important limitations that can affect their reliability and real world applicability. One major challenge is data dependency. Most models are only good on the data they are trained on. If the data is somehow biased, incomplete, or inconsistent the model will learn and change it. Too much representation of some groups can lead to unfair results. While insufficient data prevents the models from being able to learn robust patterns. Ml models also struggle when the real world environment in which they trained in changes. This causes the performance to degrade.

Another example of a limitation is the major cost that comes with training these models. These models need advanced systems. They need powerful GPU’s or TPU’s, large memory, large storage, and long training times. They also consume a significant amount of energy. These major costs limit who can develop such systems and how often they can be updated.

Another concern in explainability. Some ml models run as “black boxes.” These produce predictions without reason. Although some explanation tools exist, they often give approximates rather than a true insight into how the decisions are made. The lack of transparency is found in areas such as healthcare,finance and law. Areas where accountability and trust is needed.

Ml systems also struggle with generalization. They are effective in recognition of patterns. But are often found performing poorly on unfamiliar inputs. In real world conditions they are vulnerable to adversial attacks and edge cases. This leads to unpredictable behavior in these real world conditions.

Finally, the last factor limiting ml systems are ethical and legal constraints. Concerns about discrimination and misuse fall into regulations of GDPR. These concerns require careful oversight. Additional challenges are security risks such as data poisoning and inversion attacks on the model.
